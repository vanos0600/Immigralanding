Capabilities

Engineered to be cited, not sued.

Every layer of the system is designed for accuracy and privacy, because immigration decisions deserve more than a generic chatbot.

Hybrid retrieval

BM25 keyword search runs in parallel with ChromaDB semantic search. Legal abbreviations like "VZP" or "Bridge Label" no longer slip through the cracks.

RAG / v9.1

Privacy by design

A dedicated regex layer strips passport numbers, rodné číslo, IBAN, and phone numbers before any text reaches the language model. Zero PII leaves the building.

GDPR / Article 25

Persistent memory

Conversations are stored in Supabase via UUID sessions. Refresh the page; your history remains. Right-to-erasure handled with a single query.

SUPABASE / POSTGRESQL

Document analysis

Upload a lease or insurance contract as PDF. The agent analyses it against the 2026 legal framework in real time and surfaces what's missing or non-compliant.

PDF / PYPDF

Eleven languages

Heuristic language detection picks up Czech, Spanish, Ukrainian, Arabic, Vietnamese, and seven more. The instruction is injected after context to prevent English drift.

i18n / 11 LOCALES

Authoritative knowledge base

Eleven structured sections covering the full student immigration lifecycle, sourced from the Ministry of the Interior, Act 326/1999 Coll., frs.gov.cz, and ipc.gov.cz.

Architecture

From your question to a defensible answer.

Every message follows the same auditable pipeline. No black boxes, each stage has a single, clearly-defined responsibility.

STEP 01

Input + sanitization

Question and optional PDF enter the pipeline. Regex strips PII; heuristics detect language.

STEP 02

Context loading

Last 20 messages pulled from Supabase by session UUID for conversation continuity.

STEP 03

Hybrid retrieval

BM25 keyword search and ChromaDB semantic search execute in parallel and merge.

STEP 04

Grounded generation

Gemini 2.5 Flash answers using only the retrieved legal context and saves to Supabase.
